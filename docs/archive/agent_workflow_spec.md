# 代理人工作流規格書 (Agentic Workflow Specification) - v2

本文件定義了「AI 減脂操作系統」的四大核心代理人職能。不同於傳統的功能清單，這些工作流是以**能力領域 (Capability Domains)** 為中心，並參考 **OpenClaw** 的邏輯，實現聊天室 (LINE) 與圖文介面 (LIFF) 的深度同步。

---

## 核心架構哲學 (Architectural Philosophy)

1.  **對話為本，UI 為輔 (Chat-First, UI-Enhanced)**: 所有的操作（紀錄、修正、設定）皆可透過自然對話完成。LIFF 頁面則提供高密度的視覺反饋與儀表板。
2.  **談判式共識 (Negotiated Consistency)**: 面對模糊輸入時，Agent 透過對話發起「追問」，並在雙方達成共識後才寫入資料庫。
3.  **前瞻性修正 (Forward-looking Recovery)**: 紀錄超標後，專注於「未來三天的修正」（補償計畫），而非指責過去。
4.  **工具集約化 (Integrated Tooling)**: 如外部搜尋 (Web Search) 是解決問題的「手段」，而非獨立的工作流。

| 工作流領域 | 對應 LIFF 頁面 | 核心任務 | 主動行為 (Proactive) |

## 詳細流程說明

### Flow A: 飲食解析與共識 (Nutrition Resolution)
*   **職責**: 這是系統最頻繁執行的「主流程」，整合了原本碎裂的紀錄、追問與修正。
*   **執行路徑**:
    1.  **解析**: 使用者輸入文字/照片 -> Planner 決定解析策略。
    2.  **檢索**: 內部資料庫匹配 -> **[工具調度]** 若不確定則背景執行 Web Search。
    3.  **共識機制**:
        - 若資訊不足：發起**自然對話追問**（例如：「看起來有雞腿，請問大概多大呢？」）。
        - 若使用者修正：「剛才那個飯我只吃一半」-> 檢索 `SessionState` 並重新估算。
    4.  **寫入**: 確認後同步更新 `TodayPage`。

### Flow B: 情境化推薦 (Contextual Feed)
*   **職責**: 解決「下一餐吃什麼」的決策焦慮。
*   **執行路徑**:
    1.  **情境分析**: 檢查當前剩餘熱量、地點、時間、當日已吃內容。
    2.  **排序層**: 從 `Golden Orders` 與 `Favorite Stores` 提取候選。
    3.  **呈現**: 
        - 聊天室：給出 Top 1 最安全建議。
        - LIFF (`EatPage`)：展示完整列表與「想喝湯」、「高蛋白」等篩選器。

### Flow C: 體態指標與隱形校準 (Body Calibration)
*   **職責**: 確保預估熱量與現實體重變化掛鉤，實現「自動修正誤差」。
*   **執行路徑**:
    1.  **數據蒐集**: 使用者在對話中說「我今天 75kg」或在 `ProgressPage` 記錄。
    2.  **引擎計算**: 背景分析 14 天趨勢，計算 Reporting Bias。
    3.  **同步**: 更新 Body Goal 中的 TDEE 參數，這會影響 `TodayPage` 的每日預算上限。

### Flow D: 柔性預算與救援 (Rescue & Coaching)
*   **職責**: 當「現實」超出「計畫」時（紀錄爆卡），維持使用者的意志力。
*   **執行路徑**:
    1.  **偵測**: 紀錄一筆高熱量餐點，導致累積攝取 > 今日總額。
    2.  **方案擬定**: Planner 設計「分攤方案」（未來 2-3 天各少攝取 200kcal）。
    3.  **談判**: 對話詢問：「昨天比較豐盛，要幫你把超出的部分分三天攤平嗎？這樣明天還是可以吃飽喔。」
    4.  **生效**: 使用者同意後，動態調整未來三天的儀表板上限。

## 狀態機定義 (Agent State Machine)

| 狀態名稱 | 行為描述 | 結束/轉移條件 |
| :--- | :--- | :--- |
| `RESOLVING` | 正在嘗試解析餐點內容（可調度檢索工具） | 成功估算內容 或 需要追問 |
| `NEGOTIATING` | 已發出自然對話追問，等待使用者回覆 | 收到回覆後回到 `RESOLVING` |
| `SUGGESTING` | 正在生產餐點建議 | 使用者選擇建議內容進度紀錄 |
| `RESCUING` | 正在討論超標修正方案 | 達成共識並更新未來預算 |
