# Doc Classification Registry

This registry makes each durable repository document explicitly classifiable.

## Canonical

- `AGENTS.md`
- `docs/index.md`
- everything under `docs/specs/`
- canonical quality docs under `docs/quality/`
- active execution plans under `docs/exec-plans/active/`
- `docs/exec-plans/WORKFLOW_SLICE_REGISTRY.md`

## Supporting

- `docs/exec-plans/active/handoff/`
- `docs/exec-plans/completed/handoff/`
- `docs/exec-plans/reference/handoff/`
- `docs/governance/EXECUTION_OPERATING_MODEL.md`
- `docs/governance/CHANGE_CONTROL_GUARDS.md`
- `docs/governance/EXECUTION_SELECTION_POLICY.md`
- `docs/governance/ENCODING_POLICY.md`
- `docs/governance/SPEC_EDITING_PROTOCOL.md`
- `docs/governance/IMPLEMENTATION_PLANNING_REPLAN_PROTOCOL.md`
- `docs/governance/TASK_CHECKIN_PROTOCOL.md`
- `docs/governance/HANDOFF_CONTRACT.md`
- `docs/governance/BUILD_FILE_PLACEMENT_RULES.md`
- `docs/governance/DATA_SOURCE_POLICY.md`
- `docs/references/context_memory_architecture.md`

## Reference

- everything under `docs/references/`
- generated manifests under `docs/generated/`

## Archive / Quarantine

- everything under `docs/archive/`
- everything under `artifacts/docs-snapshots/`

## Rule

No new durable document should be added to the repo without being placeable into one of these categories.
